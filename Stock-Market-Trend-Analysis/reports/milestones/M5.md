# Milestone 5 — Real Deployment (Streamlit app)

> **Disclaimer.** This project is for educational purposes only. The forecasts produced by
> this model are not investment advice. Past performance does not predict future returns.
> The author does not recommend trading on these predictions.

**Date:** 2026-07-06 · **Status:** done, multi-agent audited · **Deliverables:** `app.py`,
`.streamlit/config.toml`, `test_app.py`, four committed artifacts (see §3).

## 1. What was built

A local Streamlit web app (`streamlit run app.py`) that replays the sealed 2025–2026 holdout:
pick a **ticker** (4), a **model** (7 prediction columns), and a **trading date**, and see the
model's next-day log-return forecast next to what the market actually did, with a HIT/MISS
direction verdict, the full holdout chart (actual vs forecast, selected date marked), and the
honest M4 accuracy table underneath. The not-investment-advice disclaimer renders in a red
error box directly under the title, before any data.

Scope follows the checklist blurb exactly: load trained artifacts + `holdout_predictions.parquet`,
ticker/date picker, forecast vs actual, disclaimer front and center, honest accuracy shown.
No training, no network calls, no "tomorrow's forecast" surface anywhere — the app can only
replay sealed history, which is the strongest structural guard against misuse.

## 2. Design decisions (ponytail)

- **No live inference for 6 of 7 models.** The app reads stored predictions from
  `holdout_predictions.parquet`. One exception proves the models are real: when the tuned
  LightGBM is selected, the saved booster is loaded and re-run on that date's features live,
  and the app asserts the fresh prediction matches the stored one (mismatch renders a red
  integrity error, not a silent pass).
- **Altair (already a Streamlit dependency) for the one chart.** No plotly/matplotlib added.
- **`st.cache_data`/`st.cache_resource` keyed on file mtime**, so regenerating an artifact
  while the server runs invalidates the cache instead of silently serving stale numbers.
- **Theme** reuses the M4 deck's light clean-finance palette (`.streamlit/config.toml`).
- **One runnable check** (`test_app.py`): Streamlit `AppTest` executes the whole script
  headlessly and fails on any uncaught exception + asserts the disclaimer is present; a second
  test re-predicts all 1,380 tuned-LightGBM holdout rows with the saved booster and asserts
  exact agreement with the stored column.

## 3. Artifacts committed for fresh-clone deployment

`data/processed/` and `models/` stay gitignored, with four small carve-outs (~660 KB total)
so `git clone` + `pip install` + `streamlit run app.py` works with no pipeline run:

| File | Size | Why the app needs it |
|---|---|---|
| `data/processed/holdout_predictions.parquet` | 96 KB | all 7 models' holdout predictions + `y_true` |
| `models/m4_holdout_scores.csv` | 4 KB | the honest-accuracy table |
| `models/lgbm_global_tuned_final.txt` | 8 KB | live-model integrity check |
| `data/processed/holdout_fe.parquet` | 548 KB | features for the live check |

If the two required files are missing anyway, the app shows a friendly error and stops
(`st.error` + `st.stop()`), not a raw traceback.

## 4. Honesty guarantees carried into the UI

- Headline metrics never color the model's call (delta_color off on prediction and verdict);
  only the actual market move is colored.
- Naive zero predicts exactly 0 every day — the app renders "no direction call" / "n/a"
  instead of fabricating a DOWN call and a directional accuracy the M4 scores table
  deliberately leaves blank.
- Every per-ticker accuracy caption prints the always-up base rate next to the model's score
  and says which is higher.
- The accuracy table shows `Days scored` and `Naive RMSE (same days)`: the LSTM/ensemble rows
  cover 1,140 of 1,380 days (60-day warm-up), and on those same days naive scores 0.02004 —
  so their lower absolute RMSE beats nothing. DM p-values vs naive shown per model.
- Chart caption states the finding: a forecast line hugging zero is honest, not broken.

## 5. Corrections (multi-agent audit, 2026-07-06)

Three parallel review agents (Model QA, Investment Researcher, code reviewer); all findings fixed:

| Sev | Finding | Fix |
|---|---|---|
| BLOCKER | The four artifacts the app needs were gitignored — fresh clone crashed with a raw `FileNotFoundError`, contradicting the docstring's "committed artifacts" claim | `.gitignore` carve-outs (`dir/*` + `!file`) + missing-file guard with `st.error`/`st.stop` |
| HIGH (×2, both agents) | Accuracy table visually contradicted its own bullet: LSTM 0.02031 / ensemble 0.02005 appeared to beat naive 0.02109, but those rows cover only 1,140 sequenced days where matched naive is 0.02004 | Added `Days scored`, `Naive RMSE (same days)`, `DM p vs naive` columns + explanatory caption |
| MED | Naive zero's constant 0 rendered as "calls DOWN" + a fabricated 45.8% directional accuracy | "no direction call" special case, n/a verdict, honest caption |
| MED | "Always predicting up would score about the same" false for below-drift models (LSTM 49.1%) | Caption now computes the base rate and says better/same/worse dynamically |
| MED | Footer claimed holdout "never used for any modeling decision" — the tuned-LGB promote/revert check was decided on it | Reworded: "opened once — for final scoring and the single documented promote-or-revert check" |
| LOW | Live LGB check silently vanished when artifacts missing; mismatch shown in a calm blue box | `st.warning` when unavailable, `st.error` on mismatch |
| LOW | Stale-cache window on long-running server | loaders keyed on file mtime |
| LOW | HIT/MISS green/red gamified single-day outcomes; "nan" cells in table; "54.3%" rounded up tuned LGB; bullet omitted the two significantly-worse models | delta_color off; `na_rep="—"`; "54.3% / 54.2%"; bullet names LightGBM-baseline p=0.003, LSTM p=0.004 |

Verified clean by the auditors: date semantics (row at date X holds the return of X+1 — checked
empirically against `holdout_fe.parquet`, no off-by-one), disclaimer placement, drift-debunk
numbers vs M4.md, cost claim, ticker category order for the live LightGBM check, and that
`with st.sidebar:` scoping is fine (Python `with` creates no scope).


## 8. Limitations

- Replays a fixed historical window; by design there is no "today" forecast and no data refresh.
- Local-only (`server.headless` set, but no auth/hosting config). Public hosting (Streamlit
  Community Cloud) would need nothing more than this repo, but is out of scope until asked.
- The live-inference proof covers only the tuned LightGBM; ARIMA/GARCH/LSTM artifacts stay
  display-only (loading TensorFlow + arch in the app adds weight for no user-visible honesty gain).
